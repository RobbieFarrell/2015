Rethinking the day view for Skyscanner

The task

Using the following scenario, perform a search on the Skyscanner website using our flights product: You and your partner, and your two children (one of whom is under 2 years old) are travelling from Edinburgh to Washington International, departing 9th November 2015 and returning 2 weeks later. You are on a budget, so Economy is the only option and you are open to non-direct flights.

We would like you to analyse this view and taking a mobile-first approach present:

  • The goals this view is trying to achieve.
  • Challenges a user may face with this view
  • Any mobile specific challenges you think this view has?
  • What you think works well
  • What metrics you would use to analyse deeper

Using this information and researching into what our competitors both within and out with the travel sector do well, we would like you to present 2-3 options for how we could improve this view and help ultimately increase conversion. For the purposes of this part of the task, static wireframes and mocks are acceptable.

Please do not be constrained by the current look and feel or the way in which data is currently presented - we are open to radical change.

Having investigated options, we would like you to choose one design to mock up fully into an HTML prototype (please note, we are not looking for production quality code, we simply would like you to be able to show this on a couple of devices – we rely on our Front End Engineers to do the heavy lifting!). This should be presented alongside rationale on how you would aim to test, validate and roll out this design.

Understanding the goals

  • The goals this view is trying to achieve
    • Show the best flight packages available to the user in price ascending order initially
    • Get users to sign up for price alerts
    • Provide the user with the option to change their dates with ease
    • Provide the user with the option to filter their results by direct or multiple stops, flight times, airlines
    • Promote the car hire and hotels section of the site (desktop only)
    • Show some ads (desktop only)
    • Help the user find the right flight for them and proceed to booking it
  • Challenges a user may face with this view
    • Comparing two different flights side by side is not possible or two different days
    • Option to save flight details/email them is not possible
    • Scrolling through pages of content can be tedious
    • It’s not possible to select outgoing or return flights individually
    • There is a lot of data to read through and some of it is unnecessary and could be hidden in a reveal more info window
    • Understanding the total cost of the flight can be difficult
  • Any mobile specific challenges you think this view has?
    • Scrolling through pages of content can involve a lot of scrolling up and down
    • Filters and sorting options are tricky to use and require extra work to scroll through different options
    • Different menus that need to be opened can be distracting
    • The excessive amount of text makes the interface quite busy
  • What you think works well
    • The page is reasonably well laid out so scanning through the results is a quick process
    • Important buttons such as filters, sort options and date changing capabilities are placed near the top
    • It’s possible to see all important details about a flight without having to click the more details button
  • What metrics you would use to analyse deeper
    • How many users used the filters and section
    • How many users returned to the home page to try a new search rather than using the 'change search' facility at the top
    • How many users clicked on the more details link for a flight
    • How long a user stayed on the page before trying a new search
    • How many users used the pagination
    • Most popular screen sizes
    • Exit pages - where did users leave the page?
    • Conversion rates

Market research

As part of the research phase I gathered a list of competitors and other website that I could draw inpsirations and market trends from. Because there is a large amount of competitors I decided to focus on 6 websites: Google Flights, Kayak, Hipmunk, Expedia, Ebookers & Edreams. I spent some time navigating all websites on desktop and mobile and then got screenshots of the key flows.

The next stage was to evaluate each site under different criteria to find out what each site did well or did not. I used the following criterias for my analysis:

  • Loading time & feedback
  • How intuitive is the UI
  • Layout of content
  • Visual design
  • Desktop vs mobile layout
  • Filters
  • Stand-out feature

User research

I picked 6 parents to fill out a survery form about Skyscanner and their opinions on the website. I got them to run through the same scenario I had to and comment on how easy it was to navigate, filter and differentiate between results and then give their feedback.

What direction to take

After my market and user research I had a clear understanding of what users would be looking for along with a list of potential ideas to explore in my designs:

  • Reduce vertical height of listings by minimising text used
  • Make it easier to modify searched terms inline
  • Add option to choose different flights rather than Skyscanner choosing both flights
  • When a user scrolls to the bottom of the page auto load more results
  • Add option to save itinerary
  • Add flexible date option to help increase chance of finding cheapest date
  • Introduce a more colourful and vibrant colour scheme
  • Recommend flights to the user
  • Have a rating for flights
  • Use a common sense filter to suggest flights which take stops, time, money etc. into consideration and don’t suggest crazy stopover times to save a few quid

Evaluating the work

I originally explored two ideas in my wireframes and both were feasible options. Ideally I'd like to fully prototype both of them and use them for A/B or usability testing. Some of the metrics I'd analyse during these tests would be:

  • How users interact with the filters? Is there any new ones they'd like to see?
  • Is there enough information displayed about the flights or is there any other important to see?
  • Do they prefer seeing both the outgoing and return flights on one screen or being able to choose both one at a time?
  • Was the pagination used or did users start a new search if they couldn't find the right flight on the first page?
  • If the user couldn't find the right flights for them, find suggestions on how they'd look for the correct one (filters, sorting orders etc).
